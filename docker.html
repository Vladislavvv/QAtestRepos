Docker

  • Разработчики используют технологии контейнеризации и виртуализации в рамках практики CI/CD для изоляции тестовых сред и упрощения развертывания тестовых инфраструктур.
  • Виртуализация: создание изолированных сред на одном физическом устройстве.
  • Для ускорения процесса используют контейнеры.
  • Контейнер: виртуальная машина без собственной операционной системы.
  • Вся работа ведется в операционной системе хоста, где запускается контейнер. Вспомогательные программы в контейнере изолированы от хоста и других контейнеров, что обеспечивает быстрый запуск и эффективность.
  • Docker: платформа для управления контейнерами, облегчающая создание, развертывание и запуск приложений в изолированных средах.
  • Основная задача Docker — упростить развертывание приложений и сократить время настройки.
  • Docker Image: набор приложений, библиотек и настроек для развертывания контейнера.
  • Docker Image создается на основе Docker-файла, в котором указываются все необходимые настройки.
  • Docker Container: контейнер, созданный на базе Docker Image.
  • Разработчик создает Image, который можно использовать для развертывания контейнеров. Тестировщики запускают контейнер для работы, при этом не имея возможности изменять Image.
